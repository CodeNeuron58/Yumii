## Prerequisites

Before meeting Yumii, ensure you have the following installed on your machine:

<FeatureList cols={3}>
  <DocsCard title="Python 3.12+" icon="rocket">
    Ensure you have the latest Python version installed on your machine.
  </DocsCard>
  <DocsCard title="Modern Browser" icon="layout-dashboard">
    Chrome, Edge, or Firefox are recommended for WebGL.
  </DocsCard>
  <DocsCard title="Microphone" icon="waveform-lines">
    Required to speak to Yumii. Ensure permissions are enabled.
  </DocsCard>
</FeatureList>
</ContentCard>

<ContentCard>
## <span style={{ color: 'var(--text-accent)' }}>1. Installation</span>

First, clone the Yumii repository to your local machine and navigate into the directory.

```bash
git clone https://github.com/CodeNeuron58/Yumii.git
cd Yumii
```

Install the dependencies using `uv` (required — do NOT use pip):

```bash
uv sync
```

<Callout type="warning" title="Warning: Do not use pip">
  Do not use `pip install`. Yumii pins `torch` to a CPU-only build to keep the installation light. Using pip will try to download the full 2 GB CUDA version.
</Callout>
</ContentCard>

<ContentCard>
## 2. Attunement (First-Time Setup)

Yumii needs an LLM to think and a Text-to-Speech provider to speak. 

Run the interactive dashboard.

If you activated your virtual environment:
```bash
# Windows
.venv\Scripts\activate
yumii

# macOS / Linux
source .venv/bin/activate
yumii
```

Or without activating:
```bash
uv run yumii
```

<Callout type="info" title="Global Execution">
  After activation, you can just type `yumii` for the rest of your terminal session. If you used the one-line installer, `yumii` always works globally without activation.
</Callout>

Since this is your first time, Yumii will automatically guide you through her **Attunement** process. You will be prompted to provide:

<FeatureList cols={2}>
  <DocsCard title="LLM API Key" icon="cube">
    We highly recommend [Groq](https://console.groq.com/) for its blazing fast speed.
  </DocsCard>
  <DocsCard title="TTS API Key" icon="face-smile">
    We recommend [ElevenLabs](https://elevenlabs.io/) for lifelike expression.
  </DocsCard>
</FeatureList>

<Callout type="success" title="Local First Security">
  Keys are stored in your **OS keychain** — Windows Credential Manager,
  macOS Keychain, or libsecret on Linux. They are **never** written to
  disk as plaintext, and **never** stored in a `.env` file. They are
  only transmitted directly to the LLM / TTS / STT provider you
  configured.
</Callout>
</ContentCard>

## Troubleshooting

<AccordionGroup>
  <Accordion title="I get a KMP_DUPLICATE_LIB_OK error">
    This is common on Windows with `faster-whisper`. The Yumii CLI automatically attempts to patch this by setting the environment variable internally, but if it persists, ensure you don't have conflicting OpenMP libraries installed.
  </Accordion>
  <Accordion title="She isn't responding to my voice">
    Check the terminal output. If you don't see `[Voice Activity Detected]`, your microphone might be muted or not set as the default system input device.
  </Accordion>
</AccordionGroup>
</ContentCard>
