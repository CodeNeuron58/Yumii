## Prerequisites

Before meeting Yumi, ensure you have the following installed on your machine:

<FeatureList cols={3}>
  <DocsCard title="Python 3.12+" icon="rocket">
    Ensure you have the latest Python version installed on your machine.
  </DocsCard>
  <DocsCard title="Modern Browser" icon="layout-dashboard">
    Chrome, Edge, or Firefox are recommended for WebGL.
  </DocsCard>
  <DocsCard title="Microphone" icon="waveform-lines">
    Required to speak to Yumi. Ensure permissions are enabled.
  </DocsCard>
</FeatureList>

## 1. Installation

First, clone the Yumi repository to your local machine and navigate into the directory.

```bash
git clone https://github.com/CodeNeuron58/Yumi.git
cd Yumi
```

Install the dependencies using `uv` (required — do NOT use pip):

```bash
uv sync
```

<Callout type="warning" title="Warning: Do not use pip">
  Do not use `pip install`. Yumi pins `torch` to a CPU-only build to keep the installation light. Using pip will try to download the full 2 GB CUDA version.
</Callout>

## 2. Attunement (First-Time Setup)

Yumi needs an LLM to think and a Text-to-Speech provider to speak. 

Run the interactive dashboard.

If you activated your virtual environment:
```bash
# Windows
.venv\Scripts\activate
yumi

# macOS / Linux
source .venv/bin/activate
yumi
```

Or without activating:
```bash
uv run yumi
```

<Callout type="info" title="Global Execution">
  After activation, you can just type `yumi` for the rest of your terminal session. If you used the one-line installer, `yumi` always works globally without activation.
</Callout>

Since this is your first time, Yumi will automatically guide you through her **Attunement** process. You will be prompted to provide:

<FeatureList cols={2}>
  <DocsCard title="LLM API Key" icon="cube">
    We highly recommend [Groq](https://console.groq.com/) for its blazing fast speed.
  </DocsCard>
  <DocsCard title="TTS API Key" icon="face-smile">
    We recommend [ElevenLabs](https://elevenlabs.io/) for lifelike expression.
  </DocsCard>
</FeatureList>

<Callout type="success" title="Local First Security">
  Keys are stored securely in your local environment file (`.env`). They are never transmitted anywhere except directly to the providers you chose.
</Callout>

## 3. Waking Her Up

Once configured, simply select **"🌸 Wake Yumi Up"** from the main dashboard menu (or run `uv run yumi wake-up`).

<StepsList>
  <div>The backend server will initialize and listen on port `8000`.</div>
  <div>A browser tab will automatically open to `http://localhost:8000/`.</div>
  <div>Allow microphone permissions if prompted by your browser.</div>
  <div>Say hello to Yumi!</div>
</StepsList>

## Troubleshooting

<AccordionGroup>
  <Accordion title="I get a KMP_DUPLICATE_LIB_OK error">
    This is common on Windows with `faster-whisper`. The Yumi CLI automatically attempts to patch this by setting the environment variable internally, but if it persists, ensure you don't have conflicting OpenMP libraries installed.
  </Accordion>
  <Accordion title="She isn't responding to my voice">
    Check the terminal output. If you don't see `[Voice Activity Detected]`, your microphone might be muted or not set as the default system input device.
  </Accordion>
</AccordionGroup>
