<ContentCard>
Yumii is driven by a lightweight **FastAPI** backend (`src/yumii/api/server.py`). Alongside the real-time WebSocket connection, Yumii serves REST endpoints for session and fact management, runtime config introspection, and mounts the static frontend.

---

</ContentCard>

<ContentCard>
## 1. Asset Mounting & Serving

Yumii bundles her frontend interface and exposes custom folders securely.

### A. Serving the WebUI
*   **Path**: `/` (root)
*   **Method**: `app.mount("/", StaticFiles(directory=webui_dir, html=True))`
*   **Description**: Serves Yumii's standard single-file client page (`index.html`) directly inside your browser.

### B. Mounting Custom Avatars
*   **Path**: `/Yumii_Avatar`
*   **Method**: `app.mount("/Yumii_Avatar", StaticFiles(directory=avatar_dir))`
*   **Description**: Exposes your custom Live2D model directories located in `~/.yumii/avatar/` to the PixiJS loading runtime.

## 2. Sessions API

Conversation session management. All session metadata is persisted in
`~/.yumii/memory/yumii.db` (`sessions` table).

| Method | Path | Description |
|--------|------|-------------|
| `GET`    | `/api/sessions` | List non-archived sessions, most recently active first. Returns `[{"id","name","created_at","last_active_at","message_count"}]`. |
| `POST`   | `/api/sessions?name=…` | Create a brand-new session. Returns `{"session_id","name"}`. |
| `POST`   | `/api/sessions/{id}/resume` | Resume an existing session by id; clears queues and resets the interrupt flag. |
| `DELETE` | `/api/sessions/{id}` | Hard-delete a session and its summary. |

---

</ContentCard>

<ContentCard>
## 3. Long-term Memory API

User facts (preferences, identity, habits, goals) stored under the
LangGraph `("facts",)` namespace in `~/.yumii/memory/store.db`.

| Method | Path | Body | Description |
|--------|------|------|-------------|
| `GET`    | `/api/facts`           | — | List all stored facts with metadata. |
| `PUT`    | `/api/facts/{id}`      | `{"fact":"new text"}` | Update the text of an existing fact. |
| `DELETE` | `/api/facts/{id}`      | — | Remove a single fact. |

</ContentCard>

## 5. CORS Configurations

Yumii's FastAPI server permits cross-origin browser connections so that
local development overlays, OBS browser sources, and other tools on the
same machine can reach the WebSocket without browser CORS errors:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

> **Security note.** This is a permissive configuration suitable for a
> local-only single-user tool. A page in **any** browser tab you have
> open can theoretically open a WebSocket to `ws://localhost:8000/ws`
> and interact with the LLM. The server is bound to `127.0.0.1` so
> LAN machines cannot reach it, and your API keys are never exposed
> to the browser, but the LLM token cost is. The v0.3.0 release will
> tighten `allow_origins` to the specific localhost / app origins
> that Yumii actually serves.

For WebSocket payload details, see the [WebSocket Protocol](/integration/websocket-protocol) page.

<Callout type="info" title="Planned for v2">
  Exposing Yumii's tool registry to external LLM clients via the
  **Model Context Protocol (MCP)** is on the roadmap for v2.0. The
  current engine is a self-contained voice → LLM → TTS loop. See the
  [MCP page](/integration/mcp-server) for details.
</Callout>
