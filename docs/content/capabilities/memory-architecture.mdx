## What v0.1.0 actually does

Yumi v0.1.0 uses **LangGraph's `InMemorySaver`** checkpointer
(`src/yumi/agent/graph.py`):

```python
from langgraph.checkpoint.memory import InMemorySaver

saver = InMemorySaver()
return workflow.compile(checkpointer=saver)
```

This means:

* Conversation history is preserved **for the lifetime of one running
  server process**.
* The checkpointer is keyed by `thread_id`. The current engine uses a
  single hardcoded thread (`yumi_session_1`), so all WebSocket clients
  share the same history.
* **Restarting the server clears all memory.** There is no disk-backed
  storage in v1.

This is sufficient for a single-session demo and matches the
prototyping posture of the project. The v1 README, `VISION.md`, and
this page are explicit about it.

## What is planned for v1.1

The v1.1 release (target: shortly after v1.0) introduces two new
storage layers:

1. **Conversation log** — every `(turn_id, role, content, timestamp)`
   tuple appended to a SQLite database at
   `~/.yumi/memory/conversations.db`. Append-only.
2. **User facts** — a `UserFacts` table
   `(user_id, key, value, source_turn_id, last_seen)` populated by a
   lightweight extraction step (probably an LLM pass at the end of each
   session, or a daily cron).

The LLM Planner (in v2.0) will read from this store. v1.1 only writes
to it.

The intent is to avoid building a vector store on day one. SQLite +
keyword matching covers the v1.1 use case. If retrieval quality becomes
the bottleneck, embeddings come later.

## Why not SQLite / Postgres today?

It would be ~150 lines of code. The reason to defer is not effort, it
is **observability**. Memory bugs only show up under real use. v1.0
shipped without memory is a clean baseline. v1.1 ships with memory and
is debugged against real conversation logs from v1.0 users.

If you want to enable persistent memory on a v1.0 build today, the
cleanest path is to fork `graph.py` and swap the checkpointer:

```python
from langgraph.checkpoint.sqlite.aio import AsyncSqliteSaver

async with AsyncSqliteSaver.from_conn_string(
    "~/.yumi/memory/checkpoints.db"
) as saver:
    app = workflow.compile(checkpointer=saver)
```

This is not officially supported in v1, but the seam is small enough
that a contributor PR is welcome.
