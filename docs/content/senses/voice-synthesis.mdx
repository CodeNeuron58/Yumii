## 1. Streaming Audio Pipeline

Yumii supports two streaming TTS providers:

### ElevenLabs
*   **Method**: Streams audio chunks in real-time. Yumii requests the raw audio bytes (typically `mp3` or `pcm_24000` formats) over an active HTTP stream.
*   **WebSocket broadcast**: As bytes arrive from the API, Yumii streams them directly to the browser via the WebSocket in raw binary audio chunks (`audio_chunk`).

### CAMB.ai
*   **Method**: High-performance alternative streaming engine, delivering high-fidelity vocal tracks in real-time.

## 2. Real-Time RMS Lip-Syncing

Once the browser WebUI receives the audio chunks, it feeds the stream into a Web Audio API `AudioContext` and routes it through an `AnalyserNode`.

To synchronize Yumii's mouth openings to the actual spoken sounds, a high-frequency `requestAnimationFrame` loop calculates the **Root Mean Square (RMS)** amplitude of the playing audio waveform in real time.

```
[ Playing Audio ] ──► [ AnalyserNode ] ──► [ Compute RMS Amplitude ]
                                                      │
  [ Live2D Parameter MouthOpenY ] ◄── [ Map 0.0 - 1.0 ] ◄───────┘
```

The mathematical computation executed in `index.html` is:

$$\text{RMS} = \sqrt{\frac{1}{N}\sum_{i=1}^{N} x_i^2}$$

```javascript
// JavaScript implementation in Yumii's WebUI
const pcmData = new Float32Array(analyser.fftSize);
analyser.getFloatTimeDomainData(pcmData);

let sumSquares = 0.0;
for (const amplitude of pcmData) {
    sumSquares += amplitude * amplitude;
}
const rms = Math.sqrt(sumSquares / pcmData.length);

// Map RMS amplitude to 0.0 - 1.0 mouth open parameter
const mouthOpen = Math.min(1.0, rms * 15.0);
live2dModel.internalModel.coreModel.setParameterValueById("ParamMouthOpenY", mouthOpen);
```

### Why this is superior:
*   **Perfect Sync**: Because mouth movement is calculated directly from the active audio wave being played through your speakers/headphones, it can never desynchronize.
*   **Expressiveness**: Whispers or quiet syllables result in small mouth openings, while loud exclamations result in fully open animations, perfectly matching the voice's emotional tone!
