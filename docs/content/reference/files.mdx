Yumii keeps all of her state in one folder in your home directory. Knowing the
layout means you can back her up, move her to a new machine, or wipe her
completely — with a file manager.

```
~/.yumii/
├── auth.json            # API keys (owner-only permissions)
├── config.json          # non-sensitive settings
├── memory/
│   ├── yumii.db         # session list (titles, timestamps, counts)
│   ├── checkpoints.db   # full conversation history per session
│   └── store.db         # long-term facts about you
├── models/              # downloaded models (source installs)
│   ├── kokoro/          # voice model + voice bank (~350 MB)
│   └── vosk/            # only if you selected the Vosk STT backend
└── avatar/              # reserved for the future Live2D companion mode
```

On Windows, `~` is `C:\Users\<you>`.

## The two config files

**`auth.json` — secrets.** Every API key, written with owner-only permissions
and atomic writes. Keys are only ever sent to the provider they belong to.
This is the same storage model tools like Claude Code use. Treat this file
like a password: don't commit it, don't screenshot it.

**`config.json` — preferences.** Plain, human-editable JSON: provider
choices, personality, model names, connected toolkits. Safe to read, edit,
or version. Every key is documented in the
[Settings Reference](/reference/settings).

Editing either file by hand works — Yumii reads them at startup, so restart
afterward. The [dashboard](/guide/dashboard) is the friendlier editor for
both.

## Practical recipes

* **Back up your companion** — copy `~/.yumii` somewhere safe. That's her
  entire memory of you, your settings, and your keys.
* **Move to a new machine** — install Yumii there, copy `~/.yumii` over,
  launch. She remembers you.
* **Reset conversations but keep settings** — delete `memory/` while she's
  closed.
* **Factory reset** — delete `~/.yumii` entirely. The next launch recreates
  it fresh (and the first-run wizard runs again).
* **Uninstalling the app never touches `~/.yumii`** — your data outlives the
  program, deliberately.

## What's *not* on disk

There is no telemetry file, no analytics ID, no cloud sync state. The
complete list of network destinations is: the LLM/STT/TTS providers you
configured, Composio if you connected apps, and (source installs only) model
downloads from GitHub on first run.
