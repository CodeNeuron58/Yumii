Source installs get a `yumii` terminal command (installer users generally
never need it — the desktop app manages everything). Run bare `yumii` for an
interactive shell with a first-run wizard, or use the subcommands directly.

## The commands you'll actually use

| Command | What it does |
|---|---|
| `yumii` | Interactive shell — wizards, session tools, and the launcher |
| `yumii server` | Run the backend headless on `127.0.0.1:8000` (what the desktop app launches) |
| `yumii wake-up` | Start the backend **and** open the browser orb |
| `yumii attune` | The first-run setup wizard (providers + keys) |
| `yumii models` | Reconfigure LLM / TTS / STT any time |

## Session & memory management

| Command | What it does |
|---|---|
| `yumii sessions` | List saved conversations |
| `yumii chat` | Start a new conversation in the browser |
| `yumii resume-chat` | Resume the most recent conversation |
| `yumii delete-session <id>` | Delete a conversation permanently |
| `yumii forget` | Wipe all long-term facts (sessions stay) |
| `yumii hitl-mode` | Set the tool-confirmation mode |

## Inside the interactive shell

Bare `yumii` opens a prompt where slash-commands (the `/` is optional) map to
the same actions:

| Shell command | Action |
|---|---|
| `/wake` | Start the server and open the browser orb |
| `/chat` | Session picker — resume, start new, or delete |
| `/resume` | Resume the most recent session |
| `/sessions` | List sessions |
| `/memory` | Browse and edit stored facts |
| `/forget` | Wipe all facts |
| `/config` | Provider/keys wizard |
| `/personality` | Switch persona |
| `/hitl` | Tool-confirmation mode |
| `/help` · `/exit` | The obvious |

<Callout type="info" title="The dashboard does all of this too">
Everything above (except launching the server itself) has a friendlier home
in the [dashboard](/guide/dashboard). The CLI remains useful for
headless setups, scripting, and source-install workflows.
</Callout>
